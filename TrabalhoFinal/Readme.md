# Formas de Contribuição Aceitas na Unidade II — Software Livre

Cada equipe e/ou aluno deverá realizar **4 contribuições distintas**, totalizando 10 pontos.
Algumas contribuições só podem ser feitas uma vez por equipe, outras podem ser
repetidas, conforme indicado.

Cada equipe criará um arquivo na pasta TrabalhoFinal relatando as tarefas feitas, e colocando links (ou arquivos) para as mesmas.

- Os bugs reports devem ser originais, bem descritos e enviados conforme as orientações do projeto.
- Traduções: Por ser uma contribuição relativamente mais simples, não pode ser repetida.
- Pequena funcionalidade: Não precisa ser aceita pelo projeto, mas deve ser demonstrada e registrada.
-	Cada equipe deverá criar um relatório do seu trabalho final baseado no modelo disponibilizado em [TrabalhoFinalExemplo.md](TrabalhoFinalExemplo.md);
-	Após finalizado o relatório do trabalho, o aluno deverá incluir também suas colaborações no [relatório da turma](trabalho-final-turma.md);

# Relatório do Trabalho Final

Nessa pasta cada equipe deve colocar um arquivo que será o relatório do Trabalho Final desenvolvido. Para tanto, devem ser realizados os seguintes passos:
   - Copiar o arquivo [TrabalhoFinal.md](TrabalhoFinal.md) e renomeá-lo para o nome do projeto que está contribuindo, por exemplo, Ubuntu.md;
   -  Preencher os dados solicitados no modelo;
   -  Separar cada tipo de trabalho, da maneira como estão especificados no documento do trabalho final, em subseções de segundo nível (##);
   -  Para cada subseção, dividir em itens (*) as tarefas realizadas, com um texto curto descrevendo-a e colocando o link para a mesma;
   -  A equipe deverá incluir suas contribuições também no relatório final da turma, que listará todas as colaborações feitas por todos em um único arquivo.

Um exemplo de como ficaria o relatório da equipe segue abaixo:

# Relatório do Trabalho Final - Ubuntu

- **Alunos:** xxxxx xxxx, xxxx xxxx
- **Nível:** Graduação
